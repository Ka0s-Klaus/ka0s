# Adding Windows Nodes to Ka0s Cluster

This guide describes how to add a Windows Server node with GPU support to your existing Linux-based Kubernetes cluster.

## Prerequisites

1.  **OS**: Windows Server 2019 or 2022 (Datacenter recommended).
2.  **Container Runtime**: Docker EE or Containerd (Containerd is now default in K8s).
3.  **Networking**:
    *   Direct connectivity to the API Server (k8-manager IP).
    *   Calico CNI for Windows enabled.
4.  **GPU**:
    *   NVIDIA Drivers for Windows installed.
    *   DirectX 12 / CUDA support.

## Step 1: Prepare the Windows Node

Run these commands in PowerShell (Administrator):

```powershell
# Install Windows Containers feature
Install-WindowsFeature -Name Containers -Restart

# Install Docker (Optional, if using Docker EE)
Install-Module -Name DockerMsftProvider -Repository PSGallery -Force
Install-Package -Name docker -ProviderName DockerMsftProvider
Restart-Computer -Force
```

## Step 2: Install Kubernetes Components

You need `kubelet.exe`, `kube-proxy.exe`, and `kubectl.exe` matching your cluster version (v1.32.x).

Download binaries from [Kubernetes Release Page](https://github.com/kubernetes/kubernetes/releases).

## Step 3: Configure Networking (Calico)

Since your cluster uses Calico (`vxlan.calico`), you must install Calico for Windows.

1.  Download `calico-windows-v3.x.zip`.
2.  Configure `config.ps1` with your Kubeconfig and Service CIDR.
3.  Run `install-calico.ps1`.

## Step 4: Join the Cluster

Use the join command generated by `kubeadm token create --print-join-command` on the master, but adapt it for Windows (usually handled by the Calico install script or manual kubelet startup).

## Step 5: Enable GPU Support

1.  **Install NVIDIA Drivers**: Download standard drivers from NVIDIA site.
2.  **Device Plugin**: Deploy the [NVIDIA Device Plugin for Windows](https://github.com/NVIDIA/k8s-device-plugin).

```yaml
kubectl apply -f https://raw.githubusercontent.com/NVIDIA/k8s-device-plugin/v0.14.1/nvidia-device-plugin.yml
```

**Note**: Windows containers with GPU support require specific base images (e.g., `mcr.microsoft.com/windows/servercore:ltsc2022`).
