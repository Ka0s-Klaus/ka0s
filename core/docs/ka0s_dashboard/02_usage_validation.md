# Guía de Uso y Validación: Gestión Visual

## 1. Flujo de Tareas
*   **Creación**: Nuevas tareas entran en `Backlog`.
*   **Desarrollo**: Al iniciar, mover a `In Progress`.
*   **Finalización**: Al desplegar, mover a `Done`.

## 2. Validación
El dashboard debe reflejar en tiempo real el estado de los Issues y Pull Requests del repositorio. Si un PR se cierra, la tarea asociada debe moverse a `Done`.
