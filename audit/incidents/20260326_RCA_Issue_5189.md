# Root Cause Analysis (RCA) - Issue #5189

**Fecha:** 2026-03-26
**Incidente:** Fallo en workflow Ka0s Agent Query
**Run ID:** 23586816619
**Estado:** Resuelto

## 🔍 Análisis del Fallo
El workflow falló en el paso `Ollama Preflight (Models)` debido a una discrepancia en el etiquetado de los modelos entre la API de Ollama y la configuración del workflow.

### Causa Raíz
1.  **Tagging Implícito vs Explícito:** El workflow requería `nomic-embed-text`. La API de Ollama devolvía el modelo como `nomic-embed-text:latest`.
2.  **Lógica de Verificación Débil:** El script de preflight comparaba el nombre exacto del modelo sin considerar el tag `:latest` por defecto.
3.  **Timeout de Pull:** Aunque el script intentó descargar el modelo, el timeout de 30 segundos era insuficiente para operaciones de pull en la red del clúster.

## 🛠️ Remediación Aplicada
1.  **Normalización de Tags:** Se actualizó el script de preflight en `kaos-agent-query.yaml` para añadir automáticamente `:latest` si no se especifica un tag.
2.  **Incremento de Timeout:** Se aumentó el timeout de `urllib.request.urlopen` de 30s a 300s para permitir descargas completas.
3.  **Validación de Salida:** Se mejoró la lógica de comparación para asegurar que los modelos se detecten correctamente tras un pull exitoso.

## ✅ Verificación
- Se modificó el workflow en la rama principal.
- Se verificó la lógica de normalización de strings mediante script de prueba local.
