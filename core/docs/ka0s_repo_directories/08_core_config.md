# `core/config/`: ficheros de control y validación

## Propósito
Configuración central para validadores/linters y ficheros de control que gobiernan checks del repo.

## Contenido típico
- Configs de linting por formato.
- “Control files” usados por workflows para validar estructura/estándares.

## Ejemplos en este repo
- `core/config/core/ka0s_c0re_files.json`
- `core/config/control-file.json`
- `core/config/control-file.yaml`
- `core/config/core/kaos-yamllint-config.yaml`
